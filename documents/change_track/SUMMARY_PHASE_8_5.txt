📊 FLUX DE DONNÉES

  config/base.yaml
  ├─ simulation.start_time = "2005-01-01T00:00:00Z"  [GLOBAL]
  └─ simulation.speed_multiplier = 1.0               [GLOBAL]
                                    ↓
  config/loader.py
  ├─ Protège start_time (pas d'override possible)
  └─ Protège speed_multiplier (pas d'override possible)
                                    ↓
  ClusterSimulator
  ├─ _start_time [modifiable via API PUT]
  ├─ _speed_multiplier [modifiable via API PUT]
  ├─ _t_elapsed_s [accumule, persiste sur reset hard]
  └─ get_snapshot() → "ts": start_time + elapsed_s
                                    ↓
  API Endpoints
  ├─ GET /simulation/config/start_time
  ├─ PUT /simulation/config/start_time
  ├─ POST /simulation/reset
  └─ WebSocket : publication snapshots avec ts
                                    ↓
  Dashboard Streamlit
  ├─ Affiche date de départ
  ├─ Date picker pour modifier
  ├─ Bouton reset complet
  └─ Reçoit snapshots en temps réel

================================================================================

✨ POINTS CLÉS DE CONCEPTION

1. Start_time éditable SANS reset du temps écoulé
   └─ Permet ajuster calendrier d'une expérience en cours

2. Speed_multiplier global + protégé
   └─ Comme start_time, garantit cohérence cross-scenario

3. Reset complet = soft + hard atomique
   └─ UNE action pour "recommencer de zéro"

4. Asyncpg pour TimescaleDB async
   └─ Dashboard Streamlit synchrone n'est pas bloqué

🎓 EXEMPLES D'UTILISATION

Exemple 1 : Rejouer une heure dans une année différente
  1. Dashboard → date picker → 2020-06-15
  2. Cliquer "✓ Appliquer date"
  3. Résultat : snapshots timestamp "2020-06-15T..." + elapsed_s

Exemple 2 : Speed global + reset complet
  1. Mettre vitesse 3600x (1 hour/sec)
  2. Attendre 5 secondes (= 5 heures simulées)
  3. Cliquer "🗑️ Reset complet"
  4. Résultat : tout à zéro + TimescaleDB vidée

Exemple 3 : Scénario → changement vitesse → reset
  1. Charger "nominal" (vitesse hérite 1.0x)
  2. Changer vitesse à 60x
  3. Changer scénario à "stress" (vitesse persiste)
  4. Cliquer reset (tout à zéro)
